# Test Plan for [demo.opencart.com](demo.opencart.com)
## Introduction
This is a demo e-commerce website used to practice software QA skills in an agile workflow.
## Purpose
Providing structured QA coverage for [demo.opencart.com](demo.opencart.com) by validating core e-commerce features (login, product browsing, shopping cart, checkout, and account management). The goal is to practice real-world QA workflows, including test planning, documentation, and execution. 
## Scope
This commit covers the login feature, including positive, negative and edge test cases. Future commits will expand to registration, product browsing, and checkout features.
## Approach
- Manual exploratory testing for initial coverage.
- Functional test cases will be documented in markdown files.
- Potential automated tests for UI actions, introduced later to practice automation.
## Tools
Microsoft Edge for browsing, GitHub for documentation and potential integration with test management tools such as Jira or Xray in the future.
## Risks/Assumptions
- Black box testing only  
- No access to backend processes.
